# คู่มือการติดตั้งและสรุปผลงาน (Installation Guide & Project Summary)

ระบบบันทึกและจัดการฐานข้อมูลประวัติการรักษาผู้เข้ารับบริการทางกายภาพบำบัด
**สหคลินิกเดชอุดม แลบ กายภาพบำบัด (ประจำปี พ.ศ. 2569)**

---

## 📌 1. สรุปการทำงานและฟีเจอร์เด่น (Summary of Implemented Features)

ระบบได้รับการพัฒนาเป็นแอปพลิเคชันแบบ Full-stack (React + Node/Express) โดยรองรับการจัดเก็บข้อมูลแบบความเสถียรสองโหมด (Dual-Mode Storage): ทำงานผ่านฐานข้อมูลระบบคลาวด์ประสิทธิภาพสูง (Supabase Cloud Database) และจะตรวจเช็กสลับลงไปบันทึกผ่านเซิร์ฟเวอร์โลคัลไฟล์ (Local JSON Storage) โดยอัตโนมัติหากยังไม่ได้ตั้งค่าคีย์เชื่อมต่อ ป้องกันระบบล่ม 100%

### ฟีเจอร์ที่ได้รับการพัฒนาสำเร็จ:
1. **ปฏิทินนัดหมายผู้รับบริการล่วงหน้า (Appointment Calendar):**
   - แสดงผลในรูปแบบตารางปฏิทินรายเดือน (Monthly Calendar View) และแบบการ์ดรายการนัดหมายทั้งหมด (Upcoming List View)
   - ดึงรายชื่อคนไข้และเลข PT แบบ **Autocomplete อัจฉริยะ** ขณะพิมพ์จองคิว ทำให้ไม่ต้องพิมพ์ข้อมูลใหม่ซ้ำซ้อน
   - แสดงสถานะเป็นสีสัญลักษณ์เฉพาะ: รอนัดรักษา (Scheduled - สีน้ำเงิน), เข้ารักษาสร็จสิ้น (Completed - สีเขียว), ยกเลิกนัด (Cancelled - สีเทา/แดง)
   - รองรับการอัปเดตสถานะทันทีในปฏิทิน และการเพิ่ม/ลบ/แก้ไขนัดหมาย

2. **ระบบแบ่งหน้าสมุดบันทึก (Pagination Control):**
   - แสดงผลแบบตารางสเปรดชีต (Spreadsheet Log) แบ่งสัดส่วนละ **10 รายการต่อหน้า** ตามที่ผู้ใช้ระบุ เพื่อการโหลดที่รวดเร็วและเป็นระเบียบ

3. **ระบบรายงานประจำเดือนอัจฉริยะ (Monthly Report Generator):**
   - สรุปตัวเลขสถิติสำคัญประจำเดือน: ยอดรายรับรวม, จำนวนครั้งที่ให้บริการ, รายการแยกผู้ป่วยรายใหม่ (New Case) / ติดตามผล (Follow-up)
   - สรุปอันดับโรค/กลุ่มอาการที่พบบ่อย (Top Diagnoses) และรายการเครื่องมือที่ใช้งานบ่อยที่สุด (Top Treatments)
   - รองรับ **การส่งออกรายงาน (Export) เป็นไฟล์ Excel/CSV** และ **สั่งพิมพ์รายงานเป็น PDF** ในหน้ากระดาษที่จัดสัดส่วนสวยงาม

4. **ระบบสลับธีมสีคลินิก (Dual UI Theme Switcher):**
   - สามารถคลิกปุ่มสลับธีมสีภาพรวมได้ 2 รูปแบบทันที: **ธีมสีเขียวรักษ์โลก (Clinical Teal)** และ **ธีมสีชมพูหวานละมุน (Romantic Pink)**

5. **ระบบความปลอดภัยและการกู้คืนข้อมูล (Security & Data Recovery):**
   - การปกป้องส่วนแก้ไขข้อมูลสำคัญด้วยรหัสผ่านผู้ดูแลระบบ (Admin PIN: `123456`)
   - ระบบสำรองข้อมูลประวัติในรูปแบบ JSON และนำเข้าไฟล์สำรองเพื่อกู้คืนข้อมูลประวัติย้อนหลังได้เสถียรที่สุด

## 🛠️ 2. คู่มือการติดตั้งระบบ (System Installation Guide)

### สิ่งที่ต้องเตรียมก่อนเริ่มติดตั้ง (Prerequisites)
- **Node.js** (เวอร์ชัน 18 ขึ้นไป แนะนำเวอร์ชัน 20 LTS)
- **npm** (ติดตั้งมาพร้อมกับ Node.js)

### ขั้นตอนการติดตั้ง (Step-by-Step Installation)

1. **แตกไฟล์และเข้าไปยังโฟลเดอร์โครงการ:**
   ```bash
   cd clinic-management-system
   ```

2. **ติดตั้งไลบรารีและ Dependencies ทั้งหมด:**
   ```bash
   npm install
   ```

3. **กำหนดค่าสภาพแวดล้อม (Environment Variables):**
   สร้างไฟล์ชื่อ `.env` ไว้ที่โฟลเดอร์หลัก (Root Directory) ของโครงการ และป้อนค่าต่าง ๆ ดังนี้:
   ```env
   # .env
   PORT=3000
   NODE_ENV=production
   GEMINI_API_KEY=your_gemini_api_key_here
   
   # คีย์เชื่อมต่อ Supabase (หากต้องการเก็บข้อมูลถาวรบนคลาวด์ ถ้าไม่ใส่ระบบจะใช้ไฟล์ JSON ในเครื่องแทน)
   SUPABASE_URL=your_supabase_project_url_here
   SUPABASE_ANON_KEY=your_supabase_anon_key_here
   ```

4. **เตรียมตารางข้อมูลบน Supabase (กรณีเลือกใช้ Supabase):**
   คัดลอกคำสั่ง SQL ทั้งหมดจากไฟล์ `/supabase_schema.sql` ไปเปิดรันในเมนู **SQL Editor** บนเว็บไซต์หน้าแดชบอร์ดของ Supabase เพื่อสร้างตาราง `patient_records`, ตาราง `appointments`, ดัชนีความเร็ว (Indexes), และนโยบายสิทธิ์การเข้าถึงแบบปลอดภัย (Row Level Security - RLS) พร้อมทั้งใส่ข้อมูลตั้งต้นของคนไข้และนัดหมายตัวอย่าง

## 🚀 3. คำสั่งสำหรับการรันระบบ (Running Commands)

### 💻 โหมดนักพัฒนาสำหรับการแก้ไของค์ประกอบ (Development Mode)
ใช้รันเมื่อต้องการเขียนโค้ดและทดสอบโปรแกรมแบบเรียลไทม์:
```bash
npm run dev
```
*ระบบจะเปิดเซิร์ฟเวอร์ Express ร่วมกับ Vite บายพาสที่พอร์ต `3000` สามารถเข้าใช้งานได้ผ่านเบราว์เซอร์ที่ลิงก์: `http://localhost:3000`*

### 📦 โหมดการใช้งานจริง (Production Build & Run)
ใช้สำหรับการขึ้นระบบงานจริงหรือโฮสต์แอปพลิเคชันบนคลาวด์:
1. **ทำการคอมไพล์โค้ดเป็นไฟล์ย่อขนาด (Build):**
   ```bash
   npm run build
   ```
2. **รันระบบเซิร์ฟเวอร์หลัก:**
   ```bash
   npm run start
   ```

## 🗄️ 4. โครงสร้างไฟล์ในโครงการ (Project File Structure)

- `src/server/app.ts` — แบ็กเอนด์ API หลัก รองรับการทำบันทึกประวัติ และบริหารจัดการคิวนัดหมาย
- `src/App.tsx` — ตัวควบคุมส่วนติดต่อผู้ใช้หลัก คุมหน้าต่างแท็บและธีมสีคลินิก
- `src/types.ts` — ตัวแปร Interface ของโครงสร้างข้อมูล `PatientRecord` และ `Appointment`
- `src/components/AppointmentCalendar.tsx` — ปฏิทินนัดหมายแบบอินเตอร์แอคทีฟรายเดือน พร้อมระบบ Autocomplete รายชื่อคนไข้
- `src/components/SpreadsheetLog.tsx` — ตารางสเปรดชีตแสดงผลข้อมูลคนไข้แบบแบ่งหน้าละ 10 รายการ
- `src/components/MonthlyReportModal.tsx` — หน้าต่างสรุปรายงานสถิติรายเดือน การส่งออก Excel และการสั่งพิมพ์ PDF
- `supabase_schema.sql` — โครงสร้างชุดคำสั่ง SQL สำหรับนำไปติดตั้งลงใน Supabase
